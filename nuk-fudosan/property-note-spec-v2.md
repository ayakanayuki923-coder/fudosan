## 1. プロジェクト概要

### コンセプト
- 不動産投資家・ゆうすけさんの個人用ツール
- **現場優先** - 外出先・手が塞がっている状況でも使える
- **スマホがメイン** - PCでも快適に使える
- 商用ではないので過剰な機能は不要。シンプル・高速・確実
- **会計ソフトは作らない** - 投資判断に必要な収支管理に徹する。確定申告・正式帳簿はfreee等の専用ソフトに任せる

### 技術構成
- **単一HTMLファイル**（index.html）
- **ホスティング**: GitHub Pages
  - URL: `ayakanayuki923-coder.github.io/fudosan/nuk-fudosan/index.html`
- **バックエンド**: Google Apps Script + Googleスプレッドシート
- **ローカル保存**: localStorage キー `fudosan_note_v2`（絶対変更禁止）
- **GASシート名**: Note / 物件 / Memo / Journal / カレンダー予定 / Learn

## 4. 機能仕様

### 4-1. 物件管理

**ステータス一覧（順序固定・全ライフサイクル対応）:**

```
【取得フェーズ】
  候補発見 → 調査中 → 内覧予定 → 内覧済 → 交渉中 → 買付済 → 契約済

【運用フェーズ】
  修繕中 → 募集中 → 運用中

【出口フェーズ】
  売却検討中 → 売却済

【離脱】
  見送り
```

※ ステータス変更理由:
- 「問い合わせ済」→ 廃止（「調査中」の一部）
- 「購入検討中」「購入済」→「買付済」「契約済」に変更（実務では買付と契約は別ステップ。買付は撤回可能、契約後は手付金が絡む）
- 「内覧予定」追加（内覧準備中の状態を明示）
- 運用フェーズ（修繕中/募集中/運用中）追加
- 出口フェーズ（売却検討中/売却済）追加

**物件フォームセクション:**
- 📍 基本情報: 日付/物件名/住所/GoogleMaps URL/種別/築年数/延床面積/土地面積/アクセス/掲載URL/ステータス
- 📜 法的・行政情報: 登記名義人/登記年月日/用途地域/建ぺい率・容積率/接道状況/上下水道/再建築/登記・権利関係メモ
- 💴 収支・数値（見込み）: 購入価格/想定家賃/近隣相場/売買取引事例/修繕費/諸費用/その他収入/その他費用
- 💰 融資情報（複数対応）: 金融機関ごとに追加可能。各レコード: 金融機関名/借入額/金利/返済期間/返済方式/月々返済額/融資ステータス/メモ/結果。否決・見送り時はアーカイブ（非表示）可能。アクティブ=「過去の打診履歴」として折りたたみ表示/自己資金
- 🚪 出口戦略: 出口方針/想定売却時期/想定売却価格/出口根拠メモ
- 📸 写真: アルバムURL/撮影状況メモ
- ❌ 見送り情報（status=見送り時のみ表示）: 見送り理由カテゴリ/見送り詳細メモ/見送り日
- その他メモ（一番下・basicセクション）

**物件詳細タブ:**
Overview / Log / Summary / Note（リッチテキスト） / Photos / 収支

**Noteタブ（リッチテキスト）:**
- ツールバー: B / I / S / H2 / H3 / • / 1. / ❝ / —
- 自動保存（入力のたびにlocalStorage + GAS同期）

### 4-2. Memo

**カテゴリ（12個・表示順固定）:**
```
物件探し / 現地調査 / 相場調査 / 法的確認 /
交渉・契約 / 融資 / 修繕・リフォーム /
入居・管理 / 収支・税務 / 売却 /
人脈づくり / その他
```

※ カテゴリ設計思想:
- 「何をした？」に対して迷わず1つ選べることが基準
- 表示順は不動産投資の行動フェーズ順（調べる→手続き→運用→出口→その他）
- フラットなセレクトボックスで表示（グループ分けUIは不要、並び順で意味が伝わる）
- 「業者」カテゴリは独立させない（業者は全カテゴリに絡むため、目的別カテゴリに吸収）

**機能:**
- 複数物件に同時紐づけ（チェックボックス）
- 選択した物件のLogに自動追加（fromMemo:true）
- 次のアクション + 予定日 → ToDoに自動登録
- URL欄あり

**カテゴリ:**
書籍 / セミナー / 現場 / ネット / YouTube / その他

**入力項目:**
- 日付（自動で今日）
- タイトル
- カテゴリ
- URL（ネタ元）

### 4-4. 音声入力（完全ハンズフリー）

**起動:** 🎤ボタンをタップするだけ

**フロー:**
```
起動 → 読み上げ「MemoかProperty、どちらに追加しますか？」
     → マイク自動起動
     → 「メモ」または「物件」と言う

[Memoフロー]
→ 「タイトルを話してください」→ 話す → 2秒無音で自動終了
→ 「内容を話してください」→ 話す → 2秒無音で自動終了
→ 確認読み上げ「タイトル:〇〇、内容:〇〇。保存するか修正すると言ってください」
→ 「保存する」→ 自動保存・終了
→ 「修正する」→ 「タイトル・内容どちらを修正しますか？」→ 選んで話し直す

[物件フロー]
→ 物件名 → 住所 → メモ（「なし」でスキップ）
→ 確認読み上げ → 「保存する」or「修正する」
→ 保存（物件名・住所・メモ・今日の日付を自動セット）
```

**無音タイマー:** 2秒無音で自動認識終了
**日付:** 全て今日の日付を自動セット

### 4-5. 利回り電卓

- パターンA/B/C... 追加・削除・名前変更
- 3セクション: 収入 / 取得費用（一時） / ランニングコスト（年間）
- 項目の自由追加・削除・単位変更（月/年/一時/%）
- 空室損失は%指定で自動計算
- 物件選択で既存値ロード、「物件に保存」で反映
- ※ 利回り電卓は**購入前のシミュレーション用**。購入後の実績記録は収支タブ（§4-8）を使う

### 4-6. カレンダー

- 全記録の日付別集約（Memo/Learn/物件Log/Journal/予定）
- 日付タップ → その日の全記録を下に表示
- 前月・翌月の数字タップ → その月に移動
- ‹ 今日 › ナビボタン
- カレンダー折りたたみ（ダッシュボード内）

### 4-7. 同期

- GAS URL + APIキー入力 → 「接続テスト&保存」
- 接続成功 → 即座に保存 + 「同期済み」表示

- 各保存操作時: 自動push

### 4-8. 収支タブ（物件詳細内）

**役割:** 購入後の実績収支を記録し、投資判断に必要な指標を自動算出する。
**位置:** 物件詳細タブの1つとして追加（Overview / Log / Summary / Note / Photos / **収支**）

**収支明細の入力:**
- [+ 収入を記録] [+ 支出を記録] ボタン
- 入力項目: 日付 / カテゴリ / 金額 / 備考 / 毎月定額フラグ
- 「詳細」を開くと accountTag（勘定科目タグ）を任意入力可能（後付けOK）

**支出カテゴリ:**
```
【取得時（一回）】
  物件購入代金 / 仲介手数料 / 登記費用 / 不動産取得税 /
  火災保険 / その他初期費用

【修繕（不定期）】
  リフォーム / 原状回復 / 設備修繕 / その他修繕

【運用（定期）】
  管理費 / 修繕積立金 / 固定資産税 / 都市計画税 /
  ローン返済 / 保険料 / その他経費
```

**収入カテゴリ:**
```
  家賃 / 共益費 / 礼金 / 更新料 / その他収入
```

**recurring（毎月定額）フラグ:**
- recurring: true のものは月初に前月と同額の明細を自動生成
- 普段の記録はイレギュラーな収支のみ（記録負荷を最小化）
- 金額変更時は手動で更新

**サマリー表示:**
```
📊 サマリー
  今月の手残り:   +XX,XXX円
  累計損益:       -X,XXX,XXX円
  投資回収率:     XX.X%（累計損益 ÷ 自己資金 × 100）
  損益分岐予測:   あとX年Xヶ月（現在ペース基準）
```

**年間収支サマリー:**
- 総収入（カテゴリ別内訳）
- 総支出（カテゴリ別内訳）
- 年間手残り / 月平均手残り

**CSVエクスポート:**
- 収支明細を日付順CSVで出力
- accountTagが入力済みの場合はCSVに含める
- 確定申告時に税理士やfreee等へ渡す用途

## 9. バグ・注意事項（再発防止）

### 絶対に再発させないバグ
1. **id型不一致** - GASからのidは文字列、生成時も文字列
   → 常に `String(i.id) === String(id)` で比較
2. **renderFilterBars nullクラッシュ** - 非アクティブページのDOM要素
   → 必ずnullチェックしてから操作
3. **pullFromSheets後にrenderMemo()を忘れる**
   → pullFromSheets完了後に renderCurrentPage() を呼ぶ
4. **saveMemoがmemo-propertyプルダウンを参照**
   → getSelectedMemoProperties() を使う
5. **JS文字列内の実際の改行**
   → 文字列リテラル内は必ず \n でエスケープ
   → innerHTML内のonclickのクォートはテンプレートリテラルで回避

### 開発ルール（絶対守る）
1. アップロードされたファイルをベースに戻さない
2. 修正JSは必ず node --check で構文確認してからHTMLに挿入
3. ファイルサイズが10KB以上減ったら異常として中断
4. バージョン変更時は「Ver XX · Property Note」と「<title>」の2箇所を更新
5. ファイル名に Ver を含める（例: fudosan-note-104.html）
6. 修正後は必ずチェックリストで主要機能の存在を確認

## 12. 設計判断の記録

本セクションは、仕様策定時に検討して決定した事項を記録する。開発時に「なぜこうなっているか」を確認する用途。

| 項目 | 決定 | 理由 |
|------|------|------|
| 写真保存方式 | 外部アルバムURL + メモ | localStorageの5MB制限。Amazonフォトで運用。Base64は不採用 |
| ID生成 | Date.now() + ランダム文字列 | 音声入力等の連続操作でミリ秒衝突を防止 |
| 会計機能 | 投資判断用の収支管理のみ | BS/PL・正式帳簿はfreee等に委譲。CSVエクスポートで橋渡し |
| accountTag | 任意入力・後付け可 | 最初は空欄でOK。確定申告時に必要に応じて埋める |
| Memoカテゴリ | 12個フラット表示 | グループ分けUIは不要。行動フェーズ順の並びで直感的に選択可能 |
| ステータス | 全ライフサイクル（13段階+見送り） | 候補発見〜売却済まで1物件を1レコードで追い続ける |
| recurring | 毎月定額の自動生成 | 家賃・ローン返済等の手入力を省略し記録負荷を最小化 |
| 利回り電卓と収支タブの関係 | 共存（役割分離） | 電卓=購入前シミュレーション、収支タブ=購入後の実績記録 |
| 融資情報 | 複数金融機関対応（配列）+アーカイブ | 複数打診が基本。否決理由の蓄積が次の打診戦略になる |
| 収支管理のツール分担 | Property Note+スプレッドシート+freee | 記録の入口はProperty Note、月次集計はスプレッドシート、確定申告はfreee等 |
